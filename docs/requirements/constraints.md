### Constraints

| ID  | Constraint  |
|---|---|
| CON-1 | The system must be accessible via a web browser on multiple platforms (Windows, macOS, Linux) and device types (desktop, tablet, mobile). |
| CON-2 | The system must be integrated with the university's Single Sign-On (SSO) for user authentication and authorization. |
| CON-3 | The system must be deployed within the `my.innopolis.university` domain and follow the university's deployment and hosting policies. |
| CON-4 | A cloud-native, service-based architecture must be used to support scalability, maintainability, and zero-downtime deployments. |
| CON-5 | The system must support role-based access control (Student, TA, Instructor, Admin) with strict data isolation between students. |
